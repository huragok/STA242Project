Facebook data visualization

Data set description

In this part of our homework, we focus on the visualization of the Facebook data for community detection in ego networks. The publication and the C++ code related to this work can be found here. The original data consists of 10 ego networks originated from 10 different root users containing 50~800 users, each labeled into a few (potentially) "circles". Also, the features of each user in the ego network is available. The goal of this work is, given a single ego network, to infer the circles and the members corresponding to each circles based on the "links" in the network and the features of each user in the ego network.


Visualization

Our visualization mainly consists of 2 parts:

  1. Visualization of the raw data, as the circles in each ego network is well-labeled.
  2. Visualization of the community detection algorithm, where the user can set the expected number of communities.
Each part of visualization focuses on 2 aspects:
  1. Find the hierarchical relationship in the original circles, i.e. which circles is fully contained in which circles.
  2. Plot the ego networks with force-directed graph (FDG), i.e. visualizing the information embedded in the ego network and the labels/community detection results.


A few more comments

Some of the ego networks may contain so many circles and users that our visualization does not work well (e.g. 0, 107, 1684, 1912). This is especially true with the FDGs, which remains an active research area. A detailed review about FDG by Stephen G. Kobourov can be found here. A few other algorithms for FDG is available in the OGDF library. Also the community detection algorithm would be computationally prohibitive for large networks. Nevertheless, for the ego network of user 414 and K = 3, the result seems nice enough.
